Project Guide

A concise guide to what HejSan became: the application, its architecture and technology, the user evaluation, and the future direction.

Overview

HejSan is a Swedish language learning prototype developed as part of the User Interface Programming and Evaluation course. The project combines a student learning experience with administrative content management tools, providing a complete educational platform prototype.

The delivered solution includes a protected student workspace, lesson-based learning flows, and an administration area for managing users, categories, lessons, learning items, and quiz questions. The project also establishes a reusable foundation for future educational, AI, and HRI-related work.

Language Learning

Students can browse lessons, review lesson details, start learning sessions, and complete guided activities.

Content Management

Teachers and administrators can manage educational content, users, lessons, learning items, and quiz questions.

Reusable Platform

The architecture and design provide a foundation for future educational applications, AI-assisted learning, and HRI research.

Interaction Styles

As part of the course requirements, HejSan supports two visual interaction styles, allowing users to personalise the learning experience through the interface.

Application

HejSan consists of two main areas: a student workspace focused on Swedish language learning and an administration workspace used to manage educational content and users.

Lesson Library

Students browse available lessons, review lesson information, and select learning activities based on their current goals.

Lesson Flow

The learning experience guides students through lesson details, lesson preparation, and the lesson player where learning activities are completed.

Progress Tracking

Student progress is recorded and presented through the protected workspace, providing visibility into learning activity and lesson completion.

Interaction Styles

Users can personalise the experience through multiple interaction styles, allowing different interface presentations while maintaining the same functionality.

Administration Workspace

Teachers and administrators manage the educational platform through a collection of content management tools.

Users

Create, edit, assign, and manage student accounts.

Categories & Lessons

Organise educational content and structure learning paths.

Learning Items

Manage vocabulary, phrases, and other lesson content used during learning activities.

Quiz Questions

Create and maintain assessment content used to evaluate student understanding.

Architecture & Technology

HejSan uses an MVC-inspired layered web architecture. The interface is implemented as HTML, CSS, and JavaScript views, API endpoints act as controller-like entry points, services coordinate application logic, repositories isolate database access, and SQLite stores the application data.

MVC-Inspired Layers

The project separates views, request handling, business logic, persistence, and data storage so each layer has a clear responsibility.

Technology Stack

The application uses HTML, CSS, JavaScript, Python, SQLite, REST APIs, and JSON to keep the data flow and implementation understandable.

Reusable UI Foundation

Shared layout files, protected workspace components, table styles, toolbars, and repeated CRUD patterns support consistency across the student and administration areas.

Course Concepts Applied

The UI applies consistency, visibility, feedback, user control, learnability, and interaction styles through repeated patterns, clear navigation, and predictable page behaviour.

User Evaluation Overview

The user evaluation was planned as a lightweight task-based usability test before the final project presentation. The goal is to evaluate whether users can understand the application, navigate between the main sections, complete the lesson flow, and provide useful feedback about the learning experience.

The evaluation focuses on practical use rather than formal research results. Participants are asked to complete realistic tasks with minimal instruction, then answer short post-test questions about usability, navigation, lesson controls, quiz completion, progress tracking, and possible improvements.

Student Test 1

A new student user creates an account, explores the application, signs up for a lesson, completes the lesson and quiz, and reviews their progress.

Student Test 2

A second new student user follows the same task flow so that the group can compare navigation, lesson flow, and usability feedback across multiple first-time users.

Student Test 3

A third new student user completes the student workflow, giving the group another independent view of account creation, lesson discovery, task completion, and progress visibility.

Demo User Test

One participant uses an existing demo account to evaluate the experience from a returning-user perspective without the account creation step.

Admin Test

One participant tests the administration workflow, focusing on how content and user management can be understood and used from the teacher or administrator side.

Evaluation Data

Each test case is stored as a separate JSON file. Placeholder data can be used before testing, then replaced with the real participant responses before the final demonstration.

Tasks Tested

Student participants are asked to create an account, spend a few minutes exploring the application, find the lesson library, sign up for a lesson, complete the lesson, change the speech speed during the lesson, complete the quiz, and view their progress.

Post-Test Questions

After completing the tasks, participants answer 1–5 scale questions where 1 means strongly disagree and 5 means strongly agree. The questions cover account creation, navigation, lesson library discovery, lesson sign-up, lesson controls, speech speed, quiz completion, progress visibility, and overall ease of use.

Open Feedback

Participants are also asked what they liked most, whether any navigation was confusing, what felt most difficult, whether any task was hard to complete, and what they would improve in a future version.

Expected Outcome

The evaluation is intended to identify strengths, usability problems, confusing areas, and improvement opportunities. The results will support the final presentation and show how feedback was collected from real use of the prototype.

Evaluation Results Summary

The evaluation was completed by five participants, including student, demo, and administrator perspectives. All participants were able to complete their assigned evaluation flow.

The results show strong overall usability, with all recorded tasks completed successfully and an average rating of 4.9 out of 5. Participants described the application as clear, straightforward, and easy to navigate.

The main improvement areas were the lesson experience, speech feedback, and making it clearer when speech input has been detected. Detailed ratings, task completion data, and open-ended responses are available in the Results tab below.

Future Direction

HejSan delivers the core learning experience, administration tools, reusable layout foundation, and interaction-style support required for the course project. Some ideas were intentionally left for future work because of the limited project scope, available time, and focus on delivering a stable prototype for the final course presentation.

The future direction below describes how the prototype could grow beyond the delivered course version while still building on the same educational platform foundation.

AI-Supported Learning

Future versions could provide personalised practice suggestions, adaptive lesson sequencing, hints, explanations, and feedback based on learner performance.

Speech & Pronunciation

Speech recognition and pronunciation feedback could support listening, speaking, and conversational Swedish practice.

Human-Robot Interaction

The lesson flow could be extended into HRI scenarios with robot tutors, embodied language practice, and controlled learning experiments.

Platform Expansion

The platform could grow with larger content libraries, additional languages, richer learning activities, teacher analytics, and stronger reusable components.
