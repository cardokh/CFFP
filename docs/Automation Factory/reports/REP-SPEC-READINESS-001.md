## Important Constraint Found

The current PostgreSQL schema script is metadata-driven but limited.

It supports:

- table creation from JSON config
- columns
- primary keys using `primaryKey: true`
- nullable flags
- column-level unique flags
- default values
- foreign-key table constraints
- `ON DELETE`

It does not currently support:

- named constraints
- table-level unique constraints
- indexes
- `ON UPDATE`

Therefore, the Organization spec must either:

1. target the current schema config capabilities, or
2. explicitly include a separate task to upgrade the PostgreSQL schema generator before generating CRUD modules.

The recommended first experiment is option 1: use current schema config capabilities so the first Organization CRUD generation can focus on CCore vertical-slice generation instead of schema-tool refactoring.

These values are not guesses in the generated code yet. They are documented proposals for approval.
