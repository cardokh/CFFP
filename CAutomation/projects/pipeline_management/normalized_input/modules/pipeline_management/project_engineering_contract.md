Project Engineering Contract
Purpose: This Project Engineering Contract defines the engineering contract for CAutomation,
CCore, and the Pipeline Management reference module.
Architecture: Pipeline 01 owns context engineering. Task 02 is the single normalization layer and
the hard minimum input-quality gate before AI usage. CAutomation and CCore define the current
automation hierarchy for the Pipeline Management reference module.
Engineering Rules: The workflow specification is the architectural contract. Implementation,
configuration, tests, documentation, manifests, and reports must stay synchronized. CAutomation
and CCore define the current automation hierarchy for the Pipeline Management reference module.
Traceability: The Project Engineering Contract references the Project Client Contract and
constrains how project-level and module-level source documents become canonical normalized
input.
Validation: Task 02 validates structural completeness, supported source format, readability,
non-empty extracted content, required terms, placeholder tokens, and cross-document references.
Testing: Unit, functional, and validation tests must exercise the PDF-first path. DOCX support
remains available as an existing normalizer but should not block progress in this iteration.
Pipeline Contract: Downstream Pipeline 01 tasks and later pipelines consume normalized
Markdown and metadata only. They must remain independent of PDF, DOCX, or any future raw
source format.
